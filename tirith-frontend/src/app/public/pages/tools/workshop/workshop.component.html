About Sprites

There are a few guidelines you should follow:
  • Sprites are an image which is shown over the skribbl avatar.
  • Sprites should follow the overall sprite style. If youre not sure if the sprite fits, contact tobeh before drawing.
  • No insider-jokes or personal rather unpopular "favourite characters". Sprites should be fun for everyone!
  • No duplicate sprites just for other colors / details. Each sprite should have its own idea behind.
  • Don't make oversized sprites. The sprite should be mostly in the avatar area.
  • No flashy / annoying animations.
  • Sprites and the Avatar are GIF animations - the frame delay looks best with 0.25s each frame.
  • GIFs dont support semi-transparent pixels. Means: Use a transparent background and only solid colors.
    Warning: Standard brushes have semi-transparent pixels, use a pixel brush!
  • The avatar has two frames. Anyway, if you draw a sprite with two frames, they will "overlap" the avatar animation.
    Like: Sometimes the sprite frame 2 is on avatar frame 1, sometimes sprite 2 on avatar 1, etc...
    This means EVERY sprite frame has to fit on every avatar frame! More about that later.

The basic workflow

A few straightforward steps how to draw your sprite:
  • Open the template file above - you'll see three groups: template (Sample avatar frames), Angel and Squid (Sample sprites)
  • Hide all groups except one of the both avatar frames. This layer will be your template.
  • Create a new layer in a new group which will be your sprite layer.
  • Draw a sprite sketch - outlines, a few colors. Then disable the activated avatar frame and view the other avatar frame.
  • You will notice the avatar moved slightly - adjust the sprite so it fits on BOTH avatar frames.
  • This can be tricky on sprites which depend on the avatar outlines - see the Angel sample sprite.
  • As soon as you finished details, shadows and outlines, copy the sprite layer.
  • You now have two layers - each is a frame. Since the sprite should have a slight movement, you'll have to edit one of the frames.
  • Simple "moving" effects can be achieved by transforming the layer - wider, higher, bigger - and editing the shadows slightly.
    You can also remove the details and re-draw them to make the new frame look a bit different.
  • Always check that the frame you're editing fits both avatar frames.
  • As soon as you think the sprite is finished, you can move to creating the gif.

Animating it

Photoshop users have a clear advantage:
  • Photoshop lets you preview & export the gif easily.
  • Open the timeline and create a frame animation
  • Create two frames and enable only one of the sprite layers on a frame
  • Set the frame delay to 0.25s & click play to preview the animation
  • To export the sprite as gif, click "File > Export > Save for Web"
If you dont have photoshop:
  • Search the web how to preview (Gimp has no preview)
  • Search the web how to export as GIF (Gimp has this ;))
  • OR: save both layers as PNG and create a gif with a random online gif creator
